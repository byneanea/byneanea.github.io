> 4. **个人的时代生存策略**：面对范式转换和重构，不仅要分清自己是“为AI打工”还是“被AI改造”，更要转变认知策略——“To Be, Then To Choose”（先在某个领域建立身份认同，再做具体选择）。不再焦虑赛道，而是去有意识地积累自己专属的行动与记忆资产。

## Document 2: AI如何颠覆软件:你能为AI打工吗?

### 内容总结

这是波太金在 2023 年 4 月写的一篇万字长文,虽然距今近 3 年,但今天读来依然极具前瞻性。文章用历史视角回顾了 PC 互联网、移动互联网、云计算三次技术变革对软件行业的影响,然后提出 AI 时代的核心判断:所有软件公司将被二分为"为 AI 打工的公司"和"被 AI 改造的公司"。文章详细分析了数据库、安全、ITSM、流程类软件(CRM/ERP)、项目管理(Jira)、图像创作(Adobe)、RPA、代码库(GitHub)等十几个细分赛道,给出了每个赛道被颠覆的风险评级和应对策略。结论是:所有公司都不缺态度,只缺认知;未来 12 个月是赛跑。

### 内容要点

- **三分法**:人调用软件(数据库/引擎) vs 软件梳理流程(CRM/ERP) vs 翻译工具(RPA/低代码)
- **核心二分法**:"为 AI 打工的公司"(底层能力提供商)vs"被 AI 改造的公司"(流程类软件)
- **数据库**:低风险,AI 会降低查询门槛但无法替代底层计算,准确性和性价比是壁垒
- **流程类软件(CRM/ERP/Jira)**:高风险,Best Practice 会被 AI 重写,交互界面将彻底改变
- **代码库(GitHub)**:从"存储工具"变成"写代码的工具",拥有"派单权",将成为 Dev 流程的核心

### 值得关注的信息

这篇文章最有价值的是**"为 AI 打工 vs 被 AI 改造"这个二分法**。它不是简单的"AI 会颠覆谁",而是给出了一个清晰的判断框架:

- 如果你的价值是**底层能力**(计算引擎、数据库、安全基础设施),AI 会让你的产品被更多人用,你是受益者
- 如果你的价值是**梳理流程、提供 Best Practice**(CRM、ERP、项目管理),AI 会重写这些流程,你的护城河会崩塌

另一个深刻的洞察是**GitHub 地位的跃迁**。过去 GitHub 只是"代码仓库",但在 AI 时代,它会变成"写代码的工具",并且掌握"派单权"——也就是说,AI 将决定谁干什么,而 GitHub Copilot 是这个派单系统的入口。这意味着代码库从配角变成了主角。

最后,文章在 2023 年 4 月就预判了"未来 12 个月是赛跑",现在回看,这个判断完全正确。那些在 2023-2024 没有快速响应的软件公司,现在已经明显掉队了。

## Document 3: Agent 真正的护城河,正在从工具转向记忆资产

### 内容总结

这篇文章由 Founder Park 发布于 2026 年 1 月,核心论点是:随着 Claude Cowork 等桌面 Agent 的爆发,行业开始意识到 RAG 和长上下文都无法真正解决 Agent 的记忆问题,独立的 Memory 层正在成为必需品。文章深入分析了记忆管理的三重硬约束:成本与性能、复用与迁移、治理与合规。然后介绍了记忆张量(MemOS)的三层记忆架构:参数化记忆(低频更新的通用知识)、激活记忆(推理过程中的短期状态)、明文记忆(可解释、可治理的外显信息)。文章还探讨了模型厂商(OpenAI/Anthropic)与第三方中立派(MemOS/Mem0)的两种路线之争,认为记忆将成为 Agent 从"助手"到"伙伴"的关键跃迁。

### 内容要点

- **RAG 的局限**:被动检索、无权重区分、无法支撑长期协作,只能完成单次召回
- **长上下文的局限**:成本高、噪声大、无法做到精确调度,不是真正的解决方案
- **三层记忆架构**:参数化记忆(稳定知识)、激活记忆(工作状态)、明文记忆(可治理信息)
- **两种路线**:模型厂商把记忆内生到入口(增加粘性)vs 第三方做独立基础设施(可迁移、可复用)
- **记忆是真正的 Moat**:一个经过长时间磨合的 Agent,其记忆资产是无法快速复制的

### 值得关注的信息

这篇文章最重要的洞察是**记忆不是功能,而是资产**。大多数人把记忆当成"Agent 的一个功能模块",但作者指出,记忆应该像"操作系统"一样,是独立的、可调度的、可治理的系统资源。

为什么这很重要?因为**模型会快速迭代,但记忆不会**。今天用 GPT-4,明天可能换 Claude,后天可能换 DeepSeek。如果记忆绑定在某个模型或某个产品里,切换成本会非常高。但如果记忆是独立的资产层,你可以"带着记忆"在不同模型之间无缝迁移。

另一个深刻的点是**记忆的分层管理**。人类大脑不是把所有信息无差别存储,而是分成短期记忆、长期记忆、情景记忆等不同层次。AI 也应该如此。MemOS 的三层架构(参数化/激活/明文)本质上是在模仿人脑的记忆机制,这比简单的"RAG + 长上下文"高明太多。

最后,文章提到的**"记忆跟人走,不跟平台走"**是一个重要趋势。就像你的数据应该属于你自己,你的记忆资产也应该属于你自己。这不仅是技术问题,也是价值观问题。

## Document 4: a16z开年观点:当供给侧跃迁,我们需要全新的思考框架

### 内容总结

这是 a16z 创始人 Marc Andreessen 和 Ben Horowitz 在 2026 年 1 月的深度对话,由 Sense AI 整理。核心论点是:**真正的创新是供给驱动的,不是需求驱动的**。当供给侧发生根本性突破(比如 AI),市场规模分析就失效了,因为新供给会创造 10 倍、100 倍甚至 1000 倍的新市场。对话中用 Substack 的投资逻辑做案例:不是因为"人们愿意为内容付费",而是因为"给创作者变现能力,会创造出今天不存在的作者和内容"。他们还讨论了媒体环境从"受控"到"解放"的转变(Elon 收购 Twitter、Substack 的坚守),以及 a16z 的核心资产——声誉的复利积累。最后,Marc 表达了对 Z 世代(Zoomers)的极度看好,认为他们是 AI 原生代,没有道德负疚感,充满动力。

### 内容要点

- **供给先于需求**:创新不是满足既有需求,而是通过创造前所未有的供给,迫使新需求出现
- **市场规模分析失效**:当供给侧发生质变,传统的市场规模分析就失效了(Uber 不只是出租车市场)
- **Substack 投资逻辑**:赌的是"有一代高质量内容因缺乏变现机制而未诞生",这是 Supply-driven market
- **杠铃型市场**:一端是海量填充物,一端是对高质量内容的巨大渴求,Substack 瞄准后者
- **声誉是复利资产**:a16z 花了十几年建立声誉,现在募资只需做两次 AMA 就完成 150 亿美元

### 值得关注的信息

这篇对话最颠覆的观点是**"供给先于需求"**。我们习惯于"需求驱动创新"的思维:用户需要什么,我们就做什么。但 Marc 指出,真正的创新是反过来的:**先创造供给,需求会随之出现**。

这个逻辑用在 AI 上就是:不要问"用户需要什么 AI 产品",而要问"AI 能创造什么之前不可能存在的能力"。比如,没人在 ChatGPT 之前说"我需要一个能写代码、写文章、陪我聊天的 AI",但当 ChatGPT 出现后,这个需求就爆发了。

另一个深刻的洞察是**"市场规模分析失效"**。传统 VC 投资时,会先做市场规模分析:这个赛道有多大?天花板在哪?但 Marc 说,当供给侧发生质变时,这些分析都是错的。Uber 不是"出租车市场",而是"按需出行市场";GPU 不是"游戏玩家市场",而是"AI 算力市场"。AI 也一样,它会创造的市场可能是现有市场的 100 倍、1000 倍。

最后,**"声誉的复利"**这个点很有启发。a16z 第一次募资(3 亿美元)花了 6 个月,开了无数次会;这次募资(150 亿美元)只做了两次 AMA 就完成了。这就是声誉的力量。在快速变化的时代,很多东西会贬值,但声誉会增值。

## Document 5: 红杉对话 LangChain 创始人:2026 年 AI 告别对话框,步入 Long-Horizon Agents 元年

### 内容总结

这是红杉资本对 LangChain 创始人 Harrison Chase 的深度访谈,由海外独角兽整理发布。核心判断是:AI 正从 Talkers 时代(ChatGPT)进入 Doers 时代(Long Horizon Agents)。Harrison 认为,Agent 的爆发经历了三个拐点:Pre-training(ChatGPT) → Reasoning(o1) → Long-Horizon Agents(Claude Code)。他详细解释了 Framework vs Harness 的区别:Framework 是无预设的抽象层,Harness 是有主见的、开箱即用的执行环境。关键能力包括 Context 压缩、文件系统权限(所有 Agent 都需要)、Sub-agents 协调。他还提出了一个争议性问题:通用 Agent 是否就是 Coding Agent?最后,他强调 Traces 成为新的"Source of Truth",Memory 是真正的 Moat,理想交互是异步管理 + 同步协作的混合模式。

### 内容要点

- **三个拐点**:Pre-training(ChatGPT) → Reasoning(o1) → Long-Horizon Agents(Claude Code)
- **Framework vs Harness**:Framework 是抽象层(无预设),Harness 是执行环境(有主见)
- **核心能力**:Context 压缩、文件系统权限、Sub-agents 协调
- **通用 Agent = Coding Agent?**:所有 Long Horizon Agent 都需要文件系统权限,写代码是最好的通用手段
- **Traces 是新的 Source of Truth**:软件的 Source of Truth 是代码,Agent 的是代码 + Tracing

### 值得关注的信息

这篇访谈最重要的洞察是**"通用 Agent 可能就是 Coding Agent"**。这是一个反直觉但深刻的观点:为什么所有 Agent 最终都要会写代码?

Harrison 的逻辑是这样的:Long Horizon Agent 需要长时间运行,Context 会很快被填满,必须有某种方式来管理 Context。最好的方式是什么?文件系统。把不重要的东西存进文件,只保留摘要在 Context 里,需要时再去查。但如果只是"虚拟文件系统",能做的事情有限。如果能写代码、运行脚本,就能做复杂得多的事情。

这意味着什么?**Coding 不是一个垂直赛道,而是所有 Agent 的底层能力**。就像人类社会中,识字不是一个职业,而是所有职业的基础能力。在 AI 时代,写代码可能也会变成这样的"基础能力"。

另一个重要的洞察是**"Traces 是新的 Source of Truth"**。传统软件开发,出了问题就去看代码,因为代码是"单一事实来源"。但 Agent 不同,大部分逻辑在模型里,不在代码里。所以你必须看 Trace,看它实际运行时发生了什么。这不仅改变了开发方式,也改变了团队协作方式——以后大家不是说"去看 GitHub",而是说"去看 Trace"。

最后,**"Memory 是真正的 Moat"**这个点和 Document 3 不谋而合。Harrison 分享了自己的经历:他的 Email Agent 用了很久,积累了很多 Memory,后来想迁移到新平台,结果丢失了 Memory,新 Agent 虽然 Prompt 和工具完全一样,但体验远不如旧的。这说明什么?**记忆是无法快速复制的资产,是真正的护城河**。

## 跨文档综合分析:涌现的主题

### 1. 识别共同模式

从这 5 篇文档中,我们可以识别出几个反复出现的模式:

**模式一:从工具到伙伴的跃迁**

- Doc 1: AI 是"深海洋流",不是工具,是底层力量
- Doc 3: 记忆让 Agent 从"助手"变成"伙伴"
- Doc 5: AI 从 Talkers(对话者)变成 Doers(执行者)

这三篇文档都在讲同一件事:**AI 的角色正在发生根本性转变**。它不再是"你用来完成任务的工具",而是"和你一起工作的伙伴"。这个转变的关键是什么?**记忆**。没有记忆的 AI 只是工具,有记忆的 AI 才是伙伴。

**模式二:底层能力 vs 上层应用的分化**

- Doc 2: "为 AI 打工的公司"vs"被 AI 改造的公司"
- Doc 3: 模型厂商(入口派)vs 第三方(基础设施派)
- Doc 5: Framework(抽象层)vs Harness(执行环境)

这三篇文档都在讲**分层**。AI 时代的竞争,不是单一维度的,而是分层的:

- **底层**:提供基础能力(数据库、引擎、记忆系统、Framework)
- **中层**:提供执行环境(Harness、工具集、协调机制)
- **上层**:提供场景化应用(Coding、SRE、Research)

关键洞察是:**底层赢家通吃,中层百花齐放,上层快速迭代**。如果你做底层,要追求通用性和壁垒;如果你做上层,要追求速度和场景化。

**模式三:供给侧革命**

- Doc 1: AI 是"深海洋流",会从底层重构一切
- Doc 4: 供给先于需求,新供给会创造新市场
- Doc 5: Agent 的能力突破会创造新的用例

这三篇文档都在讲**供给侧**。传统的思维是"需求驱动创新",但 AI 时代的逻辑是反过来的:**供给驱动需求**。当 AI 能做到以前不可能做到的事情时,新的需求就会出现。

这对创业者意味着什么?**不要去问用户需要什么,而要去创造用户想象不到的能力**。

### 2. 综合关键洞察

把这 5 篇文档的洞察综合起来,我们可以得出几个关键判断:

**洞察一:2026 是 Long Horizon Agents 元年**

Doc 5 明确指出:AI 正从 Talkers 时代进入 Doers 时代。但这不是孤立的判断,而是得到了其他文档的印证:

- Doc 1 提到 2026 年十大预测中,Agent 电商是重要方向
- Doc 3 提到 Claude Cowork 开启了桌面 Agent 的新热点
- Doc 2 虽然写于 2023 年,但已经预判了"未来 12 个月是赛跑"

综合起来看,**2026 年确实是一个关键节点**。不是因为某个模型突然变强了,而是因为**模型能力 + Harness 工程 + 记忆系统**这三者的组合,让 Long Horizon Agent 从"demo"变成了"可用的产品"。

**洞察二:记忆是真正的护城河**

这是 Doc 3 和 Doc 5 的共同结论,但可以延伸到更广的范围:

- **对 Agent 产品**:记忆让 Agent 从"助手"变成"伙伴",是用户粘性的来源
- **对垂直应用**:领域知识 + 记忆积累 = 无法快速复制的壁垒
- **对个人**:你与 AI 的协作记忆,是你的个人资产,应该跟着你走,不跟着平台走

这也解释了为什么 MemOS 这样的独立记忆系统会成为热点——因为**记忆不应该被任何一个模型或平台绑架**。

**洞察三:Coding 是新的"识字"**

Doc 5 提出了一个大胆的猜想:通用 Agent 可能就是 Coding Agent。这个观点乍听很极端,但细想很有道理:

- **技术层面**:文件系统权限是 Long Horizon Agent 的标配,而代码是操作文件系统的最佳方式
- **认知层面**:写代码的本质是"用精确的语言描述流程",这是 Agent 需要的核心能力
- **生态层面**:GitHub 从"代码仓库"变成"写代码的工具",掌握了"派单权"

如果这个判断是对的,意味着什么?**在 AI 时代,Coding 不是程序员的专属技能,而是所有 Agent 的基础能力**。就像识字不是文员的专属技能,而是所有现代人的基础能力。

**洞察四:供给侧思维 > 需求侧思维**

Doc 4 的核心论点"供给先于需求"听起来很抽象,但结合其他文档,可以变得很具体:

- **对创业者**:不要做"AI 版的 XX",而要问"AI 能创造什么之前不可能的能力"
- **对投资人**:传统的市场规模分析在供给侧革命时期会失效,要看"新供给能创造多大的新市场"
- **对个人**:不要问"我应该学什么技能",而要问"我要成为什么样的人"(To Be, Then To Choose)

这是一个**认知框架的转变**:从"满足需求"到"创造供给",从"优化存量"到"开辟增量"。

### 3. 建立理解框架

基于以上分析,我们可以建立一个理解 AI 时代的三层框架:

**第一层:技术栈(从下到上)**

- **底层**:模型能力(Pre-training → Reasoning → Long Horizon)
- **中层**:基础设施(Framework → Harness → Memory System)
- **上层**:应用场景(Coding → SRE → Research → Vertical)

**第二层:价值链(从供给到需求)**

- **供给侧**:AI 创造新能力(比如 Long Horizon Agent)
- **传导层**:新能力重构 Workflow(比如 GitHub 从仓库变成派单系统)
- **需求侧**:新 Workflow 创造新需求(比如"用 AI 写代码"成为常态)

**第三层:竞争格局(从通用到专用)**

- **通用层**:模型厂商(OpenAI/Anthropic)争夺入口
- **基础设施层**:独立玩家(LangChain/MemOS)提供可迁移的能力
- **应用层**:垂直公司(Factory/Rogo)在特定场景建立壁垒

这个三层框架的核心逻辑是:

1. **技术栈决定了能做什么**:Long Horizon Agent 的突破,来自模型 + Harness + Memory 的组合
2. **价值链决定了价值如何流动**:供给侧的突破会创造新需求,而不是满足旧需求
3. **竞争格局决定了谁能赢**:通用层赢者通吃,基础设施层多元共存,应用层快速迭代

### 4. 强调战略意义

对于不同的角色,这个框架的战略意义是不同的:

**对于创业者:**

- **如果你做底层**(模型/引擎/记忆系统):追求通用性,建立技术壁垒,目标是成为基础设施
- **如果你做中层**(Harness/工具集):追求最佳实践,快速迭代,目标是成为事实标准
- **如果你做上层**(垂直应用):追求场景化,积累记忆资产,目标是建立领域护城河

**对于投资人:**

- **看底层**:看技术壁垒,看是否有"反包"其他技术的可能
- **看中层**:看工程能力,看是否能把模型能力转化为可用的产品
- **看上层**:看场景深度,看是否能在垂直领域建立无法复制的记忆资产

**对于个人:**

- **认知层面**:从"需求侧"切换到"供给侧",从"我需要什么"切换到"我能创造什么"
- **能力层面**:Coding 可能成为新的"识字",不一定要成为程序员,但要理解代码逻辑
- **资产层面**:你与 AI 的协作记忆是你的个人资产,要有意识地管理和积累

**对于组织:**

- **战略层面**:判断自己是"为 AI 打工"还是"被 AI 改造",前者拥抱,后者警惕
- **组织层面**:记忆系统不仅是技术问题,也是组织问题,需要设计权限、隔离、共享机制
- **文化层面**:从"人调用软件"到"AI 调度人",组织文化需要适应新的协作模式

## 结论

这 5 篇文档虽然角度不同,但共同描绘了一个清晰的图景:**2026 年是 AI 从"对话"到"行动"的转折点**。

技术层面,Long Horizon Agents 的三个拐点已经完成:Pre-training(ChatGPT) → Reasoning(o1) → Long Horizon(Claude Code)。现在的 Agent 不仅能理解,能推理,还能长时间自主执行复杂任务。

产品层面,关键突破来自三个方向:Harness 工程(如何调度模型)、记忆系统(如何积累经验)、文件系统权限(如何管理状态)。这三者的组合,让 Agent 从"demo"变成了"可用的产品"。

商业层面,竞争格局正在分化:模型厂商争夺入口,基础设施玩家提供可迁移能力,垂直应用在特定场景建立壁垒。关键的判断是:底层赢家通吃,中层多元共存,上层快速迭代。

认知层面,最重要的转变是从"需求侧"到"供给侧":不要问用户需要什么,而要问 AI 能创造什么之前不可能的能力。当供给侧发生质变,市场规模分析就失效了,新供给会创造 10 倍、100 倍甚至 1000 倍的新市场。

对个人而言,核心策略是"To Be, Then To Choose":先确定要成为什么样的人,选择自然就清晰了。在快速变化的时代,身份认同比具体决策更重要。同时,要有意识地管理自己与 AI 的协作记忆,这是你的个人资产,应该跟着你走,不跟着平台走。

最后,借用老喻的隐喻:**AI 不是浪花,不是潮汐,而是深海洋流**。它会从底层重构一切规则,这不是一次技术升级,而是一次范式转变。乐观是一种选择,不是盲目,而是基于对长期趋势的判断。

2026 年,我们站在这个转折点上,接下来的 12 个月将决定谁能抓住这次供给侧革命的机会,谁会在赛跑中掉队。
